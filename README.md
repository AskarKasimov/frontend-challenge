# Кошачий пинтерест - тестовое задание

Деплой: https://askarkasimov.github.io/frontend-challenge

(или https://askarr.me/frontend-challenge - кастомный домен для Github Pages)

## Внутрянка

По стеку: React, TypeScript, Vite. Для роутинга и серверного стейта взял актуальные TanStack Router (HashRouter т.е. через #, т.к. деплой на Github Pages) и Query. Стили - SCSS модули. Ответ от апи валидируется через Zod и маппится в domain-модели

По архитектуре: я постарался сделать всё супер-качественно, но в некоторых моментах умерял свой пыл)))
Запросы к API и работа с localStorage изолированы в слое репозиториев, которые прокидываются в компоненты через useContext (DI). Сама склейка данных (сеть + проверка на избранное) делается на уровне TanStack Query (типа usecase).

По "кодовой инфре": прикрутил жесткий, но быстрый контроль кодстайла. Использую Biome, т.к. он один быстрее, чем EsLint+Prettier + настраивается гораздо проще). Для scss-модулей пришлось прикрутить Stylelint с idiomatic-order (т.к. пока Biome не работает с CSS), чтобы свойства лежали предсказуемо. А чтобы ничего кривого не улетело в репо, сделал pre-commit хук через Lefthook (выбрал его вместо Husky, т.к. проект в стиле монорепы оформлен) в связке с lint-staged

По CI/CD: деплой на Github Pages настроен через артефакты Github Actions (без билдов в мусорных ветках). Настроил кеширование yarn, проверку типов, автотесты (vitest) и линтеры перед билдом. Деплой триггерится только при обнове в мастере, а CI является необходимым условием для мерджа pull-request'ов и в dev, и в master

Еще из полезного:

- Сделал адаптивный дизайн через @media
- Бесконечный скролл сделан через Intersection Observer
- Сделал оптимистичные апдейты для лайков. Интерфейс реагирует на клик мгновенно, не дожидаясь ответа
- Написал автотест на Card.tsx (vitest + Testing Library)
- Шрифт (Roboto) не тянется линком с CDN'ок, а хостится локально через пакет @fontsource/roboto
- Сам Dockerfile написан с multi-stage: сначала билд собирается в Node-образе, а затем отдается легковесному Nginx. В итоге никаких сурсов, node_modules в проде - образ весит минимум и чисто раздаёт статику

## Запуск

Весь код фронта лежит в директории `catty-frontend`. Сделал так, чтобы сымитировать монорепу

Запускать будем всё через Docker-Compose

1. Создать файл `.env` по примеру `.env.example`

2. Запустить:

   ```bash
   docker-compose up --build
   ```
